Actors & Phân quyền Multi-Tenant

🛡️ System Admin

  • Đăng nhập qua appsettings.json (fallback) hoặc DB
  • Truy cập toàn bộ dữ liệu (IsSystemAdmin = true → bypass Global Filter)
  • Quản lý: POI, Audio, Translation, Tour, QR, Analytics, Users
  • Dịch AI tự động (Gemini API)
  • Phê duyệt/Từ chối POI (Approval Workflow + Glassmorphism Modal)
  • Xem doanh thu đăng ký, Reset mật khẩu Merchant

🏪 Owner (Agency / Chủ Quán)

  • Sử dụng Blazor Server Web Dashboard (Cookie Auth).
  • Sở hữu Dữ liệu cá thể hoá thông qua Global Query Filter theo AgencyId. Không xem được doanh thu hay POI của nơi khác.
  • Hệ thống CMS JS Heartbeat âm thầm ghi nhận thời điểm Online mà không phá vỡ liên kết Server WebSocket.
  • Trình Quản lý POI duyệt/từ chối 1 vòng với hộp chuông Notification xịn xò.

🧳 Tourist (Du khách Toàn Cầu)

  • Sử dụng Mobile App (.NET MAUI) với thiết kế Dark Glassmorphism Premium.
  • Xác thực Bảo Mật (JWT): Hành khách bắt buộc Đăng ký/ Đăng nhập tài khoản qua Email để dùng App, lưu Token bằng MAUI SecureStorage.
  • Tự động kích hoạt cơ chế Heartbeat Timer 5s: App tự "thở" nhịp tim lên Server bằng HTTP khi đang mở Foreground.
  • Live Kick-out: Bị văng thẳng ra màn hình Đăng Nhập kèm Xoá sạch JWT nếu bị Admin cắm cờ `IsLocked`.
  • Toàn quyền xài GPS bản đồ, TTS đa tuyến dịch thuật tự động.

Chapter 01.5 (NEW)

Hệ Thống Tracking Nhịp Tim & Live Kick-out (Bảo Mật)

VinhKhanh Tour sở hữu kết cấu Middleware Security Đánh Chặn (Intercept) thời gian siêu thực — kết nối chặt chẽ giữa 3 đầu Frontend MAUI, Blazor Server, và API.

❤️ 5-Second Heartbeat Pulse

Khác với việc log trạng thái truyền thống chỉ khi submit form, ứng dụng Mobile liên tục truyền một sóng âm (GET request) cực nhẹ về /api/auth/heartbeat. Bộ lọc MemoryCache Throttle Server tự động ghìm luồng để không làm tắc nghẽn PostgreSQL mà vẫn ghi xuất cực chuẩn xác thuộc tính LastActiveAt của từng người.

⛓️ Web CMS Ping Hook

Với Admin và Chủ quán sử dụng Blazor Server (công nghệ WebSockets không truyền HTTP định kì), hệ thống được "cấy ghép" bằng một Javascript Worker đơn giản ở layout gốc trỏ về /api/heartbeat-ping, ép buộc luồng đứt gãy phải hiện hình cho UserActivityMiddleware đếm nhịp!

⚔️ Live Kick-out Guard (Trục xuất Tức thì)

Admin chỉ cần bấm "Khóa tài khoản" trong CMS. Ở ngay mili-giây tiếp theo (Max ping 5s), Heartbeat của người bị phạt đập vào Firewall. Server nhả HTTP_401 (Unauthorized). Logic Application Lifecycle của Điện Thoại bắt lấy lỗi này, huỷ lập tức Storage JWT, giết bộ đếm và thổi bay User về giao diện Đăng nhập kính mờ!

Chapter 04

GPS Geofencing + Auto Narration

Hệ thống 3 giai đoạn xử lý GPS: lấy vị trí → phát hiện vùng → phát thuyết minh. Tối ưu pin và chống spam.

Giai đoạn 1 — GPS Polling (MapPage)

DispatcherTimer mỗi 5 giây → GetLocationAsync(Best, 5s timeout) → fallback GetLastKnownLocationAsync(). Quyền: LocationWhenInUse. Android 14+: Background Service.

Giai đoạn 2 — Geofence Detection

Haversine distance tới mỗi POI. Nếu ≤ poi.Radius (mét) → trigger. Ưu tiên: khoảng cách gần nhất + poi.Priority cao nhất.

Giai đoạn 3 — NarrationEngine

Cooldown 2 phút / POI (bypass nếu manual click). CancelCurrentNarration() nếu chuyển POI. Cascading Locale → TextToSpeech.SpeakAsync(). Song song ghi UsageHistory + đẩy CMS.

Circle Color States: Xám = chưa trigger, Vàng = đang phát thuyết minh, Cyan = được chọn từ list. Mỗi trạng thái ánh xạ qua UpdateMapCirclesState() với alpha transparency.

Chapter 05

Audio / TTS — Hybrid 2-Tier

2-Tier Hybrid Audio

Tier 1
Pre-recorded MP3 (0ms latency) Mobile App: Audio Element / MediaPlayer phát file MP3. CMS Listen App: HTMLAudioElement.play(). Điều kiện: AudioUrlVi hoặc AudioUrl{Lang} không null.
Tier 2
On-Device TTS (0ms, always available) Mobile: TextToSpeech.SpeakAsync(DisplayTtsScript, SpeechOptions) — dùng giọng đọc hệ điều hành. Web: SpeechSynthesisUtterance (Browser API). Fallback cuối cùng, offline-ready.

TTS Cascading Locale Algorithm

Lấy mã ngôn ngữ hiện tại → GetLocalesAsync() → 1. Khớp mã 2 chữ (vi) → 2. Khớp prefix (vi-VN) → 3. Khớp tên English → 4. Fallback ID

Chapter 06

Authentication & Merchant Onboarding

🔐 Cookie Auth (CMS)

  • Claims: Name, Role (admin/owner), AgencyId, ForcePasswordChange
  • BCrypt.Net-Next: Hash & Verify mật khẩu
  • Fallback: AdminAccount từ appsettings.json
  • Cookie Scheme: CookieAuthenticationDefaults
  • Login path: /login, Logout: /Account/logout

📝 Đăng ký Merchant

  • Trang /register — public (không cần login)
  • Form: Họ tên, SĐT, Địa chỉ, Email, Tên quán, Mật khẩu
  • Atomic Transaction: IDbContextTransaction
  • Tạo: AgencyId → POI → AppUser → AuditLog
  • Phí: 500,000 VNĐ (ghi vào AuditLog)
  • Rollback toàn bộ nếu 1 bước fail

🔄 Đổi mật khẩu + Force

  • ForcePasswordChange = true → ép redirect /change-password
  • MainLayout kiểm tra claim trên mỗi request
  • Sau đổi PW: update hash + ForcePasswordChange = false
  • Tự động logout → Re-login với PW mới
  • Admin có thể Reset PW bất kỳ user nào

Merchant Registration Flow

Tourist truy cập /register → Điền form + Validation → Kiểm tra SĐT trùng → BEGIN TRANSACTION → Tạo AgencyId + POI + User → COMMIT → Hiển thị tài khoản

Chapter 08

Offline-First Sync (ApiService)

Mobile App hoạt động Offline-First: hiển thị dữ liệu SQLite local tức thì, đồng bộ CMS Server ngầm. Analytics cũng đẩy ngầm, an toàn khi mất mạng.

2 Chiều Đồng bộ

📥
POI Sync (Server → Local) GET /api/poi → so sánh từng POI → Add/Update local → Delete POI orphan (không còn trên server). Silent fail nếu offline.
📤
Analytics Sync (Local → Server) POST /api/analytics/sync-usage batch → nếu 200 OK → xoá records local đã sync. SemaphoreSlim(1,1) tránh trùng lặp. Timeout 60s, retry khi mở app lần sau.
Data Safety: UsageHistory luôn ghi vào SQLite local trước, sau đó mới đẩy CMS ngầm. Kể cả khi mất mạng 100%, dữ liệu vẫn an toàn trong SQLite và sẽ sync khi có mạng trở lại.

sequenceDiagram

    participant TTS as OS TextToSpeech

    T->>MP: Chuyển sang Tab Map

            TTS-->>T: Phát âm thanh thuyết minh
            NE->>NE: Ghi UsageHistory
        else Ngoài bán kính
            MP->>MP: Circle = Xám
        end
    end

UC5. Đồng bộ Offline-First

sequenceDiagram
    participant API as ApiService
    participant Repo as Local SQLite
    participant CMS as CMS Server
    participant Lock as SemaphoreSlim

    Note over API,CMS: Phân hệ đồng bộ ngược (Local -> Server)
    API->>Lock: WaitAsync(0)
    alt Lấy được Lock
        API->>Repo: GetUsageHistoryAsync()
        Repo-->>API: List UsageHistory chưa sync
        API->>CMS: POST /api/analytics/sync-usage (Batch)
        alt Server 200 OK
            CMS-->>API: Success Response
            API->>Repo: Delete các records đã sync
        else Network Error
            API->>API: Giữ nguyên DB, Retry lần sau
        end
        API->>Lock: Release()
    else Đang Sync
        API->>API: Skip (Tránh gửi lặp)
    end

sequenceDiagram
